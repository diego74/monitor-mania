# Plan: Sistema de Ramificación Dinámico Configurable por Firestore

## Contexto
El usuario solicita convertir el sistema de ramificación actual (hardcoded) en un sistema completamente configurable vía Firestore. Actualmente:
- Las secciones/preguntas de ramificación están hardcoded en `compositeTestConfig.js`
- Las condiciones de activación de módulos y ramificaciones están en código
- No se puede configurar el orden de las secciones
- No hay UI para gestionar estas configuraciones

## Objetivos
1. ✅ Mover todas las configuraciones a Firestore (secciones, módulos, reglas de ramificación)
2. ✅ Permitir configurar condiciones de activación de módulos y ramificaciones
3. ✅ Permitir ordenar secciones (drag & drop) - excepto gateways
4. ✅ UI administrativa completa para gestionar todo
5. ✅ Mantener retrocompatibilidad con datos existentes

## Arquitectura Propuesta

### Firestore Schema

#### 1. Colección: `composite_config` (documento único)
```javascript
{
  enabled: true,
  branchingEnabled: true,
  maniaThreshold: 1.5,
  hypomaniaThreshold: 2.2,
  depressionThreshold: 1.5,
  activeBranchingRules: ['mixed_state', 'val_energy', ...],
  customRules: [
    {
      ruleId: 'custom_1',
      name: 'Regla personalizada 1',
      active: true,
      order: 10
    }
  ],
  moduleConfig: {
    mania: { enabled: true, order: 20, condition: { type: 'score', dimension: 'maniaScore', operator: '>=', threshold: 2.2 } },
    hypomania: { enabled: true, order: 30, condition: { type: 'score', dimension: 'maniaScore', operator: '>=', threshold: 1.5, max: 2.2 } },
    depression: { enabled: true, order: 40, condition: { type: 'score', dimension: 'depressionScore', operator: '>', threshold: 1.5 } }
  },
  updatedAt: timestamp
}
```

#### 2. Colección: `composite_sections`
```javascript
{
  _id: 'string',
  name: 'Módulo manía',
  type: 'module', // module | confirmation | branch | gateway
  category: 'mania', // mania | hypomania | depression | psychosis | confirmation | branch
  role: 'patient', // patient | caregiver | both
  order: 20,
  enabled: true,
  condition: {
    type: 'score',
    dimension: 'maniaScore',
    operator: '>=',
    threshold: 2.2
  },
  triggersFrom: ['gateway'], // secciones que pueden activar esta
  questions: [
    {
      id: 'pt1',
      question: '¿Cuántas ideas nuevas...?',
      options: [...],
      measures: 'flight_of_ideas',
      order: 10,
      overrides: {
        patient: { question: '...', options: [...] },
        caregiver: { question: '...', options: [...] }
      }
    }
  ],
  createdAt: timestamp,
  updatedAt: timestamp
}
```

#### 3. Colección: `branching_rules` (existente, extender)
```javascript
{
  _id: 'mixed_state',
  name: 'Estado mixto',
  type: 'mixed',
  active: true,
  order: 10,
  condition: {
    type: 'score',
    dimension: 'depressionScore',
    operator: '>=',
    threshold: 3
  },
  condition2: {
    type: 'score',
    dimension: 'maniaScore',
    operator: '>=',
    threshold: 3
  },
  question: '¿Sentís que estás agitada y triste al mismo tiempo?',
  options: ['No', 'Un poco', ...],
  measures: 'mixed_features',
  triggersModule: 'psychosis', // opcional
  triggersSection: 'psychosis', // nuevo
  role: 'both', // patient, caregiver, both
  updatedAt: timestamp
}
```

## Componentes a Implementar

### 1. SeccionesAdmin.js (nuevo)
- Lista de secciones con drag & drop
- Orden por tipo (gateway, module, branch, confirmation)
- Toggle enabled/disabled
- Botón "+ Nueva Sección"

### 2. SeccionForm.js (nuevo)
- Formulario para crear/editar secciones
- Configuración de condiciones (score, answer)
- Configuración de preguntas por role
- Orden numérico

### 3. BranchingRulesAdmin.js (extender)
- Integrar con el nuevo sistema de secciones
- Condiciones avanzadas (AND/OR)
- Múltiples triggers

### 4. ModuleConfig.js (nuevo)
- Configuración de umbrales (mania, hipomanía, depresión)
- Condiciones de activación
- Orden de aparición

### 5. DynamicCompositeEngine.js (nuevo)
- Motor que construye el test dinámicamente desde Firestore
- Evalúa condiciones en tiempo real
- Ordena secciones según configuración

## Consideraciones Técnicas

### Drag & Drop
- Usar `react-dnd` o `@dnd-kit/core`
- Orden numérico en Firestore (no arrays)
- Actualización optimista + rollback

### Condiciones
- Soportar múltiples tipos: score, answer, composite
- Operadores: >=, >, <, <=, ==, between
- Condiciones compuestas (AND/OR)

### Rendimiento
- Cache local de configuración
- Suscripción en tiempo real solo para admin
- Lectura única para usuarios

### Seguridad
- Rules de Firestore para admin
- Validación de esquemas
- Sanitización de inputs
